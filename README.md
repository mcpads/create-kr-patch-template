# 레트로 게임 한글 패치 프로젝트 템플릿

한글 패치 프로젝트를 시작할 때 저작 자산을 git에서 제외하고, 조사와 제품 빌드를 분리하며, 위험한 이진 쓰기를 검증하기 위한 최소 골격이다. 플랫폼·구현 언어·CLI·데이터 형식은 대상 조사 뒤 프로젝트가 정한다.

한글화 판단 방법은 [create-retro-game-kr-patch](https://github.com/mcpads/create-retro-game-kr-patch)가 설명한다. 기존 저장소에 동등한 구조와 검사가 있으면 이 템플릿의 디렉터리나 참고 코드를 덧씌우지 않는다.

## 자동으로 막을 것

공통 검사는 실제 입력만으로 잘못됐다고 확정할 수 있는 경우에 한정한다.

- 지원 원본의 크기와 강한 해시가 다르다.
- Expected Write의 원본 byte가 다르거나, 쓰기가 겹치거나, 보호·비대상 영역을 바꾼다.
- 최종 diff에 등록되지 않은 변경이나 설명되지 않은 크기 변화가 있다.
- 기계어 영역을 데이터로 쓰거나, 어셈블리 원천과 대상 ISA 검산 없이 기계어를 생성·이동한다.
- 무변경을 주장한 변환이 선언한 전체 경계를 byte 단위로 재현하지 못한다.

[`conformance/`](conformance/)와 Rust 참고 구현은 이 범위만 다룬다. 프로젝트에 같은 검사가 이미 있으면 교체하거나 그대로 사용한다.

## 판단과 증거로 남길 것

다음은 선언끼리 맞는지만 검사해서는 부족하며, 대상 조사와 HITL 판단이 필요하다.

- 번역 모집단이 정말 완전한지
- PoC 표본이 전체 범위를 대표하는지
- 번역·용어·UX가 배포 가능한 품질인지
- 사람 검수가 충분했는지
- 실행 시나리오가 변경의 위험 범위를 충분히 다뤘는지
- 어떤 입력이 실제로 수동 파생물인지

프로젝트는 이 판단의 현재 상태와 근거를 기록하고, 대상에서 기계적으로 확정된 하위 조건만 테스트로 만든다. 예를 들어 알려진 ID의 누락은 자동 검사할 수 있지만, 알려진 ID 목록이 전체 모집단이라는 결론은 조사 근거가 필요하다. 실행 증거도 정확한 산출물에 연결하되 해시 일치만으로 실행 범위가 충분하다고 판정하지 않는다.

제품 빌드는 사용자가 공급한 원본과 버전 관리되는 번역·설정·소스에서 처음부터 다시 실행돼야 한다. 이를 범용 그래프 선언으로 대신하지 말고, 대상 프로젝트의 실제 빌드 진입점을 깨끗한 상태에서 실행해 확인한다.

## 조사와 제품 빌드

조사와 PoC는 `research/`에서 자유롭게 수행한다. 확정한 변환을 채택하면 PoC 출력이나 수동 중간 파일을 복사해 제품 입력으로 삼지 않고, 순수 원천에서 같은 결과를 만드는 구현을 주 빌드에 연결한다. 채택한 폰트·인코딩·재삽입·코드 변경은 미완료 번역을 원문으로 남기더라도 하나의 개발 빌드에서 함께 검증한다.

주요 문자 집합은 출처·버전·라이선스를 확인할 수 있는 기성 글꼴에서 시작한다. 필수 기호 누락이나 실제 UI에서 확인한 UX 요구가 있을 때만 필요한 글리프를 국소 보완한다.

## 저장소 구성

- `.gitignore`: 원본 이미지, 전체 출력, 작업 파일, 실행 증거와 에뮬레이터 상태 제외
- `assets/`: 번역·글꼴·직접 작성한 그래픽 원천
- `config/`: 대상별 설정
- `src/`: 번역 인코딩, 글리프 변환, 배치와 빌드 구현
- `tests/`: 대상에서 확정한 실패 조건과 통합 빌드 검증
- `research/poc/`: 제품 빌드와 분리된 조사 프로그램
- `docs/`: 조사 결론, 재현 절차와 QA 기록
- `conformance/`: 구현 언어와 무관한 최소 이진 안전 반례
- `reference/rust/crates/patch-guard/`: 교체 가능한 참고 구현
- `adapters/emucap/`: 선택 가능한 실행 검증 연결 규칙

빈 디렉터리는 역할을 제안할 뿐 고정 구조가 아니다. 기존 프로젝트나 선택한 언어에 더 자연스러운 경계가 있으면 책임만 보존하고 바꾼다.

## 참고 구현 확인

```bash
cd reference/rust
cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --locked
```

원격 CI는 기본으로 만들지 않는다. 로컬 검사는 에이전트가 실행하며, 외부 실행 비용이 드는 자동화는 사용자가 선택한다.

원본 ROM·디스크 이미지와 패치된 전체 이미지는 커밋하지 않는다. 직접 작성했거나 배포 조건을 확인한 입력과 원본을 재현하지 않는 배포 산출물만 추적한다.
