# 번역 자산 구성

이 디렉터리는 번역 자산을 검토 가능한 여러 파일로 유지하기 위한 골격이다. 실제 직렬화 형식과 필드명은 대상 프로젝트가 정하되, 파일의 역할은 다음과 같이 나눈다.

- 색인은 지원 원본, 번역 범위, 조각의 상대 경로와 순서, 현재 검토 승인을 소유한다.
- `segments/`의 각 조각은 장면, 소비자, 스크립트 구역처럼 함께 이해하고 검토해야 하는 의미 단위를 소유한다.
- 빌드는 색인에 열거된 조각만 읽는다. 디렉터리 전체 탐색이나 첫 일치로 제품 입력을 고르지 않는다.

조각은 임의의 행 수나 배치 크기가 아니라 독립적인 변경 이유를 기준으로 나눈다. 다른 장면과 별개로 재추출·재번역·검토할 수 있는 범위는 분리하고, 문맥을 함께 봐야 하는 짧은 대화까지 한 줄씩 흩뜨리지 않는다.

빌드 전에는 모든 조각을 합친 범위에서 안정 ID의 유일성, 보호 원문과 raw byte, 제어 토큰과 참조의 유효성을 확인한다. 검토 승인은 색인과 색인이 가리키는 모든 조각의 내용 해시에 묶으며, 어느 파일이든 바뀌면 승인을 다시 요구한다. 빌드 보고서는 색인과 사용한 조각을 각각 식별해 하나의 거대 파일로 합친 임시 산출물을 원천처럼 취급하지 않는다.
