# 번역 자산 구성

이 디렉터리는 번역 자산을 검토 가능한 여러 파일로 유지하기 위한 골격이다. 실제 직렬화 형식과 필드명은 대상 프로젝트가 정하되, 파일의 역할은 다음과 같이 나눈다.

- 색인에는 지원 원본, 선언한 대상 범위, 모집단 근거와 조사 완료 여부, 조각의 상대 경로와 순서, 현재 검토 승인을 둔다.
- `segments/`는 장면이나 스크립트 구역처럼 함께 이해하고 검토해야 하는 의미 단위별로 나눈다.
- 빌드는 색인에 열거된 조각만 읽는다. 디렉터리 전체 탐색이나 첫 일치로 제품 입력을 고르지 않는다.

조각은 임의의 행 수나 배치 크기가 아니라 독립적인 변경 이유를 기준으로 나눈다. 다른 장면과 별개로 재추출·재번역·검토할 수 있는 범위는 분리하고, 문맥을 함께 봐야 하는 짧은 대화까지 한 줄씩 흩뜨리지 않는다.

빌드 전에는 모든 조각을 합친 범위에서 안정 ID의 유일성, 보호 원문과 raw byte, 제어 토큰과 참조의 유효성을 확인한다. 검토 승인은 색인과 색인이 가리키는 모든 조각의 내용 해시에 묶으며, 어느 파일이든 바뀌면 승인을 다시 요구한다. 빌드 보고서는 색인과 사용한 조각을 각각 식별해 하나의 거대 파일로 합친 임시 산출물을 원천처럼 취급하지 않는다.

현재 알려진 모집단은 지원 원본에서 재현하는 추출 단계나 검토된 대상 명세에서 얻고, 그 ID가 번역 조각에 모두 존재하는지 비교한다. 번역 조각에 이미 들어 있는 ID를 다시 모집단으로 삼아서는 누락을 검출할 수 없다. 아직 번역하지 않은 단위는 지우지 않고 원문 유지 상태로 남긴다. 더 발견될 단위가 있다면 조사 중임을 명시하고, 릴리스 전에는 단위가 더 늘지 않는다는 근거로 선언 범위를 확정한다. 범위 확정은 게임 전체를 번역한다는 뜻이 아니라 메뉴, 본편 대사처럼 처음 선언한 대상의 경계가 정해졌다는 뜻이다.

## 번역 상태의 생명주기

각 의미 단위는 미번역, 작업 중, 교차 검토 필요, 사람 판단 필요, 완료 가운데 현재 상태를 표현할 수 있어야 한다. 프로젝트는 다른 이름이나 더 세분된 상태를 써도 되지만, 빌드에서 원문을 유지할지 현 번역을 사용할지의 선택과 번역 완성 상태를 하나의 값으로 합치지 않는다. 미완료 단위도 개발 빌드에서 원문 유지 또는 명시적으로 선택한 초안으로 참여할 수 있다.

상태는 필요에 따라 이전 단계로 돌아간다. 원문 범위, 제어코드, 용어, 글리프 공급이나 배치 조건이 바뀌면 영향을 받은 단위를 다시 검토 대상으로 돌린다. 각 단위는 안정 ID 하나로 추적하고 같은 단위의 상태를 갱신한다.

대규모 사람 검수는 초벌 번역이 요구하는 글리프 집합과 대표 문장을 현재의 글꼴·인코딩·배치 설계로 표시할 수 있음을 확인한 뒤 시작한다. 이 확인에 실패하면 해당 설계의 미해결 제약으로 기록하고 글리프·인코딩·배치를 다시 조사한다. 게임이나 플랫폼에서 번역 표시가 불가능하다고 단정하지 않는다.

릴리스 승인은 단위별 완료 상태와 별개다. 선언한 범위가 완료된 뒤 사람이 현재 범위와 내용을 승인하며, 번역 내용, 보호 정보, 알려진 ID 모집단이나 조사 완료 여부가 바뀌면 승인은 무효가 된다. 영향을 받은 단위를 어느 상태로 돌릴지는 변경 범위에 따라 별도로 판단한다.
