# 폰트 자산 구성

이 디렉터리는 기성 글꼴 원천과 프로젝트가 직접 보완한 글리프를 분리한다. 파일 형식과 채택할 글꼴은 대상 프로젝트가 정하지만, 주요 문자 집합은 출처·버전·라이선스를 확인할 수 있는 기성 글꼴에서 공급한다.

## 기성 글꼴 원천

`sources/`에는 배포가 허용된 글꼴 원본 파일이나, 사용자가 별도로 공급할 파일을 식별하는 프로젝트 입력을 둔다. 채택한 원천마다 다음을 확인한다.

- 글꼴 식별자와 정확한 버전·강한 해시
- 출처와 사용·수정·재배포 조건
- 공급하는 문자 범위와 적용할 렌더링 경로
- 결과 픽셀을 바꾸는 크기·기준선·이진화·여백 등의 변환 입력

빌드는 명시적으로 선택한 원천만 읽는다. 디렉터리 자동 탐색이나 첫 일치로 글꼴을 고르지 않는다. 원본 파일을 저장소에 배포할 수 없다면 파일 대신 식별 정보와 공급 조건을 추적하고, 사용자가 제공한 정확한 입력을 빌드에서 검증한다.

문자 매핑의 형식과 위치는 프로젝트가 정한다. 다만 번역 인코딩과 글리프 변환이 함께 읽는 명시적인 제품 입력이어야 한다.

## 국소 보완

`overrides/`에는 기성 글꼴이 제공하지 않는 필수 문자·기호나, 실제 UI에서 확인된 판독성·상태 구분 등의 UX 요구를 해결하는 글리프만 둔다. 각 보완에는 문자 또는 안정 ID, 적용 렌더링 경로, 직접 제작이 필요한 이유와 검토 근거를 연결한다.

예를 들어 채택한 기성 글꼴에 `…`가 없다면 해당 기호만 보완한다. 이를 이유로 본문 한글까지 직접 제작하지 않는다. 국소 보완으로 필요한 문자와 UX를 충족할 수 없고 일관된 전체 자형 설계가 필요하다는 근거가 확인된 경우에만 전체 글꼴 제작을 검토한다.

## 제품 빌드 연결

글꼴·보완·변환·문자 매핑 가운데 프로젝트가 직접 공급하는 입력은 검토 가능한 빌드 입력으로 둔다. 대상 원본이나 다른 제품 입력에서 얻는 값은 같은 주 빌드에서 다시 만든다. 래스터화·패킹 결과와 매핑 검증 보고서는 빌드 때 다시 만들며, PoC 출력이나 수동 변환 결과로 대체하지 않는다.

빌드 보고서는 실제로 사용한 원천과 보완 입력을 식별하고, 필요한 문자에 원천·보완·매핑이 없으면 결과를 만들지 않는다. 글리프 변환은 번역 인코딩과 같은 문자 매핑을 사용하며, 두 결과를 배치와 같은 최종 빌드에 합쳐 검증한다.
