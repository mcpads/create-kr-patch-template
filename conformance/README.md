# Conformance cases

이 디렉터리는 특정 언어나 프로젝트 파일 형식을 정하지 않고, 다른 구현도 같은 결과를 내야 할 반례를 기록한다.

- `write-plan.json`: 영역, 쓰기 소유권, 기대 원본 바이트와 기계어 검증
- `product-graph.json`: 순수 원천과 제품 단계의 입력·산출물 연결
- `build-mode.json`: 알려진 번역 모집단, 개발 빌드의 자율성과 릴리스 후보의 완료 조건
- `roundtrip.json`: 무변경을 주장하는 변환 경계의 바이트 동일성
- `runtime-evidence.json`: 실행 증거와 검증 대상 산출물의 연결

JSON은 프로젝트가 채택해야 할 저장 형식이 아니라 구현 간 결과를 비교하기 위한 테스트 자료다. `reference/rust/crates/patch-guard/` 통합 테스트가 모든 항목을 읽어 기대한 승인·거부 결과와 일치하는지 확인한다.

새 실패를 막는 규칙을 추가할 때는 다음을 함께 바꾼다.

1. 가장 작은 재현 반례
2. 기대 결과와 실패 이유
3. 참고 구현
4. 우회 진입점까지 포함한 테스트

성공 사례나 플랫폼 지식은 이곳에 넣지 않는다. 이곳의 항목은 대상이 달라도 그대로 성립하는 빌드 안전 판정이어야 한다.
