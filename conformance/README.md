# Conformance cases

이 디렉터리는 실제 byte만으로 승인·거부를 확정할 수 있는 최소 반례를 둔다.

- `write-plan.json`: 원본 전제, 쓰기 겹침, 영역 구분, 기계어 검산과 전체 diff
- `roundtrip.json`: 무변경을 주장한 전체 경계의 byte 동일성

JSON은 프로젝트의 저장 형식이 아니라 다른 구현이 같은 실패를 막는지 비교하는 테스트 자료다. Rust 참고 구현은 이 항목을 읽어 기대 결과와 대조한다.

번역 모집단의 완전성, 사람 승인, PoC 대표성, UX와 실행 검증 범위는 여기에 추가하지 않는다. 그런 항목은 자기 선언의 일관성만 확인해서는 참인지 알 수 없다. 대상 조사로 기계적인 실패 조건이 확정되면 해당 프로젝트의 테스트로 남긴다.

공통 항목은 플랫폼과 프로젝트 판단 없이도 같은 입력에서 항상 같은 결과가 나오는 경우에만 추가한다.
