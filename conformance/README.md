# Conformance cases

이 디렉터리는 실제 byte만으로 승인·거부를 확정할 수 있는 최소 반례를 둔다.

- `write-plan.json`: 원본 전제, 쓰기 겹침, 영역 구분, 기계어 조립·연속 해석·재조립과 전체 diff
- `roundtrip.json`: 무변경을 주장한 전체 경계의 byte 동일성

JSON은 프로젝트의 저장 형식이 아니라 다른 구현이 같은 실패를 막는지 비교하는 테스트 자료다. Rust 참고 구현은 이 항목을 읽어 기대 결과와 대조한다.

공통 항목은 플랫폼과 프로젝트 판단 없이 같은 입력에서 항상 같은
결과가 나오는 경우에 한정한다. 번역 범위·사람 승인·UX·실행 범위는 각
프로젝트가 판정하고, 그 과정에서 확정한 기계적 조건은 프로젝트 테스트에
둔다.

참고 코어는 대상 검증기가 제공한 명령 경계와 재조립 결과만 검사하며,
선언한 ISA 프로필의 완전성은 대상 프로젝트 테스트가 맡는다.
